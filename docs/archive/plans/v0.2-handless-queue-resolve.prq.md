# Feature: Handless Queue-and-Resolve Model

## Overview

Full sim + UI rewrite to match the clarified v0.2 design in
[RULES.md](../RULES.md). Removes the hand; makes drawing an action;
introduces queue-and-resolve end-of-turn; adds retreat, closed ranks,
dominance, and tangibles vs intangibles.

**Superseded plans**: `v0.2-spec-alignment.prq.md` (partial work is
stashed on `stash@{0}`; salvageable pieces noted per-epic below).

## Spec Highlights (from RULES.md)

- **No hand.** Every card goes directly from deck to a turf on play.
- **Drawing is an action** (1 of 3/4/5 depending on difficulty & turn).
- **Opener is 5 actions**; subsequent 3 (easy–med), 4 (hard), 3 (nightmare+).
- **Turn advances only after BOTH players end.**
- **Queue-and-resolve**: draw / play / retreat immediate; strikes and
  recruits queue; resolution phase fires once both ended.
- **Dominance** = power + tangibles + affiliation loyalty — determines
  resolution order.
- **Intangibles** (counter, subtract, bribe-out, loyalty-flip,
  self-attack) fire first in resolution, in rarity order.
- **Placement rules**: modifier can't be first or last card of stack;
  unplayable drawn modifier is lost at end-of-turn.
- **Retreat**: spend 1 action, flip current top face-up permanently,
  swap any face-up card in to be the new top. No cap.
- **Closed Ranks**: end turn with no active tough → face-down top,
  can't strike, +defense inverse to difficulty.
- **Top dies → next card exposed, modifiers under stay face-down**.
- **Information asymmetry**: own fan fully face-up; opponent fan
  mostly face-down except revealed by retreats / resolution.
- **Card management** (not deckbuilding): enable/disable cards,
  priority slider 1–10 biases draw shuffle (soft influence).

### Epic B — Sim Core (Maya)

- [ ] P1: Rewrite `src/sim/turf/environment.ts`:
  - Remove `drawPhase` / `tickRound` / `advanceTurn`.
  - `stepAction` handles: `draw` (move top of deck to a turf via auto-placement? No — see below), `play_card` (from deck, not hand), `retreat`, `direct_strike`/`pushed_strike`/`funded_recruit` (queue only, don't resolve), `end_turn` (flip `turnEnded`, enforce placement cleanup).
  - **Drawing semantics**: `draw` action moves the top of the deck into a "pending placement" slot on the player (add `PlayerState.pending: Card | null`). Next action must place it or retreat will fail.
    - If drawn card is a tough: next `play_card` must be this card.
    - If drawn card is a modifier: can be tucked beneath current top via `play_card`; if no tough in play and budget exhausted, lost at end-of-turn.
  - **Placement rules** enforced at play time: modifier onto empty turf rejected (already enforced); modifier as last card of stack auto-pushed under a tough if one follows, or lost at end-of-turn if not.
  - **Queue strikes**: `strike actions` only append to `player.queued`; they cost an action but do not mutate opponent state.
  - **End turn**: sets `player.turnEnded = true`; if both players ended → call `resolvePhase` (new) → reset `turnEnded` on both → advance turnNumber → reset action budgets → clear queues.
- [ ] P1: Create `src/sim/turf/resolve.ts`:
  - `resolvePhase(state: TurfGameState): void`
  - Step 1: compute dominance per queued action — `attacker.P + tangibles + loyaltyBonus` vs `defender.R + tangibles + loyaltyBonus`.
  - Step 2: sort queued actions by dominance (dominant side first; ties favor defender).
  - Step 3: for each queued action in order, run intangibles (counter / subtract / bribe-out / loyalty-flip / self-attack) in rarity order (legendary → rare → common). Intangibles can cancel/redirect the action.
  - Step 4: if the action survives intangibles, run tangible combat via `resolveStrike` (existing logic, but flip top face-up at start, preserve modifiers face-down).
  - Step 5: after all resolutions, mark any modifier-only-top turfs: if turf is in closed ranks at this point, apply the difficulty-inverse defense bonus (so future resolution ticks use it if re-attacked mid-phase — usually zero).
  - Step 6: cleanup — unplayable pending modifiers discarded.
- [ ] P1: Rewrite `src/sim/turf/game.ts`:
  - `runTurn` now takes `actionsA: TurfAction[], actionsB: TurfAction[]` and feeds both sides their actions in interleaved order (doesn't matter — neither mutates opponent state until resolution).
  - `isGameOver` remains winner detection.
- [ ] P1: Rewrite `src/sim/turf/attacks.ts`:
  - Split into `queueStrike(player, action)` (pure — validates and stores) and `resolveStrikeNow(attackerTurf, defenderTurf, kind, state)` (called from `resolvePhase`).
  - `resolveStrikeNow` flips defender top face-up (always), runs dominance + tangibles, kill/sick/bust outcomes, transfers modifiers on kill (keep them face-down on attacker's side unless attacker chooses retreat-reveal).
  - Salvage Maya's stashed attack work — tangible ability hooks live here.
- [ ] P1: Update `src/sim/turf/board.ts`:
  - `addToStack(turf, card, faceUp = true)` — wraps in `StackedCard`; own-side defaults face-up; opponent-view defaults face-down.
  - `setTopFaceUp(turf)` — for retreat and resolution reveal.
  - `flipCardFaceUp(turf, stackIdx)` — permanent.
  - `positionPower` / `positionResistance` — unchanged math, but unwrap `StackedCard.card`.
  - Salvage Lucas's stashed loyal-stack bonuses + buffer consumption.
- [ ] P1: Add `retreatAction(state, turfIdx, targetStackIdx)` — validate targetStackIdx points to a face-up tough (or auto-flip top), swap positions so it becomes top; mark the previous top permanently face-up at its current stack position.
- [ ] P1: Add `closedRanksBonus(difficulty)` from `turf-sim.json`.

### Epic D — Abilities — Tangibles vs Intangibles (Kira)

- [ ] P1: Rewrite `src/sim/turf/abilities.ts` (salvage Maya's stashed 429 LOC where applicable):
  - Each ability declares `kind: 'tangible' | 'intangible' | 'both'`.
  - Tangible effects: flat power/resist bumps, sick-on-hit, bruiser precision-ignore, shark/ghost target override.
  - Intangible effects:
    - `counter` (weapon: cancels one kill outcome) — consumes this modifier
    - `subtract` (drug: reduces opponent power during this resolution)
    - `bribe` (currency abilities: buy off if sum ≥ threshold and loyalty match)
    - `loyaltyFlip` (currency+tough ability: flip the attacker tough onto defender)
    - `selfAttack` (drug ability: redirect queued strike onto attacker's own turf)
  - Export `runIntangiblesPhase(state, queuedAction) → 'proceed' | 'canceled' | 'redirected'`.
  - Export `applyTangibles(attackerTurf, defenderTurf) → { atkBonus, defBonus, targetOverride }`.
- [ ] P1: Wire intangible/tangible hooks into `resolvePhase`.
- [ ] P2: Update card JSON `abilities` fields to tag tangible/intangible where meaningful.
