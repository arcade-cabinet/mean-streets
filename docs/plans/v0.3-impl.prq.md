# Feature: v0.3 Full Implementation

## Overview

Implement the v0.3 design captured in `docs/RULES.md` v0.3 and validated in
`docs/plans/v0.3-paper-playtest-2.md`. Replaces the v0.2 handless
queue-resolve sim with single-lane progression, damage tiers, heat +
raids, Black Market, Holding, Mythic pool, and AI-parallel progression.

**Scope**: sim + ECS + UI + data migration + AI + tests. Full
iteration, NOT a diff on v0.2. Archive v0.2 paths where the v0.3
model makes them obsolete.

## Constraints

- pnpm only (no npm/yarn)
- Biome lint (no ESLint)
- Max 300 LOC per file (strict)
- `pnpm run lint` clean, `pnpm run test` green (node + DOM + browser), `pnpm run build` clean.
- E2E must pass on desktop-chromium, iphone-14, pixel-7, ipad-pro-landscape.
- Squash-merge via a single PR against main.
- Release gate re-baselines expected; target Medium AI-vs-AI winrate 45-55%.

## Epic A — Types & Shape (owner: Rex)

Rewrite `src/sim/turf/types.ts` for v0.3. Keep file under 300 LOC.

### Additions

- `Rarity` — add `'mythic'` to the existing union.
- `CardInstance` — `{ cardId, rolledRarity, unlockDifficulty }` — the collection unit.
- `Turf.isActive: boolean` — true for the current engagement turf only.
- `Turf.reserveIndex: number` — position in progression queue (0=active).
- `ToughCard.hp: number` and `ToughCard.maxHp: number` — HP tracking.
- `ToughCard.woundClampRatio(): number` — current HP / maxHP.
- `StackedCard.owner: string` — tough id that owns this modifier (for modifier-swap tracking).
- `QueuedAction` — extend with `'send_to_market'`, `'send_to_holding'`, `'modifier_swap'`, `'black_market_trade'`, `'black_market_heal'`.
- `TurfGameState.heat: number` — shared scalar [0, 1].
- `TurfGameState.blackMarket: ModifierCard[]` — pool of displaced mods.
- `TurfGameState.holding: { A: ToughWithMods[], B: ToughWithMods[] }` — toughs in cop custody.
- `TurfGameState.lockup: { A: ToughWithMods[], B: ToughWithMods[] }` — seized toughs with return turns.
- `TurfGameState.mythicPool: MythicCard[]` — 10-entry pool, subtract assigned.
- `TurfGameState.mythicAssignments: { [mythicId]: 'A' | 'B' | null }` — assignment tracking.
- `WarStats` — per-turf seizure tracking for victory rating.
  - `seizureTurnCounts: { seizedTurfIdx, seizedOn, wasWinningSide }[]`

### Removals

- `hasStruck` (queued model replaces).
- `turnSide` (parallel turns).
- `phase: 'combat'` → now `'action' | 'resolve'`.

2. **`src/sim/turf/attacks.ts`** — 300 LOC max
   - `queueStrike(player, action)` — unchanged signature.
   - `resolveStrikeNow(state, queued)` — expanded:
     - Apply tiered damage from §7.
     - HP accounting + wound clamp.
     - On kill: modifier → Black Market (seize reconciliation).
     - Call `runIntangiblesPhase` (abilities.ts) for priority chain.

3. **`src/sim/turf/resolve.ts`** — 300 LOC max
   - `resolvePhase(state)` — two-pass:
     - Pass 1: compute dominance per queued, sort by dominance, tie-break toward defender.
     - Pass 2: priority chain per queued (affiliations → currency → drugs → weapons).
     - After all strikes: seize reconciliation.
     - Raid **FIRST** then combat (if raid fires, no combat).
   - `raidPhase(state)` — **new**. Heat check, raid fires, market wiped, face-up tops → Lockup.

4. **`src/sim/turf/heat.ts`** — **NEW**. 150 LOC max.
   - `heatContribution(turf)` — compute per-turf heat.
   - `totalHeat(state)` — sum both sides, apply relief abilities.
   - `raidProbability(heat, difficulty)` — `heat² × coefficient`.

5. **`src/sim/turf/market.ts`** — **NEW**. 200 LOC max.
   - `sendToMarket(state, side, toughId)` — tough + mods to market.
   - `tradeAtMarket(state, side, offeredMods, targetRarity)` — pyramid trade.
   - `healAtMarket(state, side, toughId, offeredMods)` — partial/full heal.
   - `returnFromMarket(state, side, toughId)` — free end-of-turn return.
   - `wipeMarket(state)` — raid effect.

6. **`src/sim/turf/holding.ts`** — **NEW**. 200 LOC max.
   - `sendToHolding(state, side, toughId)` — tough to Holding pool.
   - `holdingCheck(state, side)` — per-turn probabilistic cop check.
   - `bribeSuccess(tough, amount)` — formula per §8.4.
   - `returnFromHolding(state, side, toughId)` — end-of-turn return if no check.
   - `lockupProcess(state)` — advance lockup turn counter; release at end of duration.

7. **`src/sim/turf/environment.ts`** — 300 LOC max
   - `stepAction` extended with new actions (`send_to_market`, `send_to_holding`, `modifier_swap`, `black_market_trade`, `black_market_heal`).
   - `retreat` unchanged.
   - `close_ranks` available only when no face-up deeper tough exists (guard check).

8. **`src/sim/turf/game.ts`** — 300 LOC max
   - `runTurn(match, actionsA, actionsB)` — unchanged in shape.
   - `promoteTurfOnSeizure(match, side)` — **new**. Called when active falls.
   - `isGameOver(match)` — existing logic; both-zero = draw.

9. **`src/sim/turf/env-query.ts`** — 300 LOC max
   - `enumerateLegalActions(state, side)` — includes all new actions, respects holding check + active-turf-only rules.
   - `createObservation` — add heat, marketSize, holdingCount, lockupCount, pending.

### Public API of `heat.ts`

```ts
export interface HeatBreakdown {
  fromRarity: number;
  fromCurrencyConcentration: number;
  fromLaunder: number;       // negative (relief)
  fromLowProfile: number;    // negative (relief)
  total: number;             // clamped [0, 1]
}

export function computeHeat(state: TurfGameState): HeatBreakdown;
export function raidProbability(heat: number, difficulty: DifficultyTier): number;
```

### Public API of `market.ts`

```ts
export function sendToMarket(state, side, toughId, pending?: Card[]): void;
export function tradeAtMarket(
  state,
  side,
  offered: ModifierInstance[],
  targetRarity: Rarity
): ModifierInstance | null;
export function healAtMarket(
  state,
  side,
  toughId: string,
  offered: ModifierInstance[]
): boolean;
```

## Epic C — Abilities (owner: Kira)

Extend `src/sim/turf/abilities.ts` + `abilities-effects.ts`.

### Updates

- Scale all `TangibleDelta` by rolled rarity multiplier (×1.0 / 1.15 / 1.3 / 1.5 / 1.7).
- `applyTangibles` — respect HP clamp on toughs.
- `runIntangiblesPhase` — add new intangibles:
  - **bribe** — now probabilistic, see §10.3. Outcome depends on dice.
  - **launder** — `-0.1 heat` per turn (relief).
  - **low_profile** — `×0.5 heat` on owner tough.
  - **clean_slate** — one-shot Mythic ability, resets heat to 0.
  - **strike_two / chain_three** — hit N cards per resolution.
  - **strike_retreated** — target requires face-up-via-retreat tough.
  - **immunity** — mythic, cannot be sent to Holding.

### New ability index entries

- PATCHUP (common drug) — +1 HP/turn for owner.
- FIELD_MEDIC (rare tough ability) — +1 HP/turn for any wounded on same turf.
- RESUSCITATE (legendary drug) — one-shot full heal for owner; then consumed.
- THE_MEDIC (legendary tough) — once-per-war full heal to any wounded target.
- LAUNDER (legendary currency) — heat relief.
- LOW_PROFILE (rare drug) — heat relief scoped to owner.
- CLEAN_SLATE (Mythic) — instant heat wipe.
- STRIKE_TWO (Mythic Shadow) — +1 strike per resolution.
- CHAIN_THREE (Mythic Warlord) — +2 strikes per resolution.
- STRIKE_RETREATED (Mythic Ghost) — target face-up deeper tough.
- IMMUNITY (Mythic Magistrate) — can't be sent to Holding.
- NO_REVEAL (Mythic Phantom) — skips resolution reveal.
- ABSOLUTE (Mythic Reaper) — always deals min wound damage on glance.
- INSIGHT (Mythic Informer) — reveals opponent heat breakdown.
- BUILD_TURF (Mythic Architect) — +1 reserve turf on play.
- TRANSCEND (Mythic Fixer) — immune to affiliation penalties.

- Pre-war phase: AI runs its own `collectionCurationPlanner` to decide merges, enable/disable, and priority per card.
- Same scoring heuristics as match-time but applied to collection state.

## Risks

- **Scope**: 1-2 week estimate assumes parallel specialists.
  Manual scope is 4-6 weeks.
- **Benchmark collapse**: introducing heat, market, holding
  simultaneously will likely crash the AI-vs-AI winrate. Plan on
  2-3 tuning iterations before the release gate passes.
- **Mythic balance**: 10 hand-authored cards with game-warping
  abilities. Each needs individual validation. Simulation can't
  tune these — only paper playtesting.
- **UI complexity**: the single-lane board + market + holding +
  heat UI is genuinely new territory. Plan on 2-3 design iterations
  before polish lands.
